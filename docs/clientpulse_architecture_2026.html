Recommended MVP Stack

LayerChoiceWhy
UI Streamlit Fastest for file upload, chat, tables, and demo video.
Backend Python + FastAPI optional Easy parsing, scoring, agent tools, and local demo deployment.
Agent orchestration OpenAI Agents SDK Best code-first choice for multi-agent workflows, handoffs, approvals, and state.
Model calls Responses API Good for model calls with tools, File Search, and structured outputs.
Knowledge retrieval OpenAI File Search Handles uploaded contracts, SOWs, QBR notes, and status reports with semantic and keyword search.
Deterministic tools Function calling with strict schemas Reliable scoring, SLA comparison, scope checks, and action generation.
Storage Local files + SQLite Simple for hackathon. Move to Blob, SharePoint, Postgres, or Delta later.
Build assistant Codex Useful for generating UI, parser code, tests, and refactoring fast.

Enterprise Version Stack

LayerChoiceWhy
UI React + ChatKit or Workspace Agent Better user experience, sharing, permissions, Slack, and repeatable workflows.
Knowledge OpenAI File Search, Azure AI Search, or Databricks Vector Search Depends on where HCL wants data governed and stored.
Connectors MCP and enterprise app connectors Optional Jira, ServiceNow, SharePoint, CRM, finance, and CLM integrations.
Data platform Postgres, Databricks Delta, or enterprise warehouse Needed for portfolio analytics and long-term risk tracking.
Governance SSO, RBAC, audit logs, traces, guardrails Required for contracts, client data, and manager approval workflows.
HCL alignment AI Force if available Matches HCL’s enterprise agentic AI platform direction, RAG, integrations, observability, and governance.
MVP

What to build first

  • Upload contract, SOW, status report, SLA CSV, QBR notes.
  • Extract obligations, risks, blockers, and SLA values.
  • Run risk and confidence scoring tools.
  • Generate contract risk report and action plan.
Do not overpromise

What stays optional

  • Jira, ServiceNow, Azure DevOps, CRM, and finance integrations.
  • Automatic ticket creation or client email sending.
  • Legal approval or final contract interpretation.
  • Historical churn model unless real renewal data exists.
Winning angle

Why this is strong

  • Works with incomplete project data.
  • Uses real manager workflow inputs.
  • Flags risks to HCL with evidence.
  • Turns analysis into recommended actions.
Final architecture position Use OpenAI Agents SDK and Responses API as the main build path. Use OpenAI File Search for uploaded project documents, function calling for deterministic checks, Codex to accelerate development, and Workspace Agents only as an optional enterprise channel. Keep Agent Builder out of the core architecture because it is not the safest 2026 foundation.
ClientPulse AI gives HCLTech managers a practical agentic system to analyze contract risk, project health, renewal readiness, missing evidence, and next-best actions using realistic manager-uploaded project data.
